Launch Control XL (Oiko)

An extended version of Bitwig’s factory Launch Control XL script (Mk2). It keeps the factory template behaviour intact and adds clean handling for user templates plus a dedicated arp layer taken from Eric Ahrens' rhbitwig script.

Setup

  • Add the controller entry “Launch Control XL by Oiko Audio” in Bitwig’s Controller settings. It uses the standard LCXL MIDI ports.
  • If Bitwig’s automatic detection adds the factory LCXL entry, remove it and keep only the Oiko entry to avoid duplicate binding.
  • Power-cycle the LCXL and select Factory Template 1 before adding if you see any odd behaviour.

Factory Templates (8–15)

  • Unchanged from the factory script: knobs/faders control sends/device targets per template, transport/track buttons work as usual.
  • LED feedback follows the selected factory template.

User Templates (1–7)

  • All MIDI is passed through to Bitwig; the script does not repaint knob LEDs, so your Novation Editor colors are preserved.

Arp Layer (User Template 8)

  • User Template 8 engages the arp layer (from rhbitwig). Factory layers are disabled; the layer targets the selected Bitwig Arpeggiator device on the first track. If you select a track without an Arpeggiator, LCXL LEDs go dark.
  • Template select: only User Template 8 is supported for the arp layer.
  • Octave range: dedicated control to pick 1–4 octaves.
  • Pitch offset: Row 1 + faders sum to ±24 semitones. LED colors show aggregate offset (yellow=0, red for negative, green for positive; brighter at ±12/±24).
  • Velocity: Row 2 sets per-step velocity (brighter green = higher velocity).
  • Gate length: Row 3 sets per-step gate (brighter yellow = longer; full CCW mutes the step/LED off).
  • 2×8 button matrix (mode-dependent):
    • Default (no mode button): Row 1 = Step Skip, Row 2 = Step Count.
    • Track Select Left: Row 1 = Step Count, Row 2 = Global Velocity presets.
    • Track Select Right: Row 1 = Step Count, Row 2 = Global Gate presets.
    • Other mode buttons (Device/Mute/Solo/Record) mirror the rhbitwig mappings for pitch/velocity/gate submodes and pattern/quantize helpers.

Notes

  • Tested in Bitwig 5.3 (API 24 runtime) and 6.0 Beta (API 25). Build against extension-api:24.
  • Debug/telemetry messages are prefixed with [LCXL] in the Bitwig controller console.
